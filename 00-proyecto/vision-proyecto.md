# Visión del proyecto KOVA A Kantar – Discovery

Este proyecto tiene como objetivo entender a fondo la realidad operativa
y tecnológica de A Kantar (unidad de negocio de ESmart) para:

- Definir el alcance funcional y técnico de KOVA A Kantar como tenant.
- Derivar requerimientos genéricos para KOVA CORE (producto Horeca).
- Diseñar una arquitectura de conocimiento gobernada basada en:
  - Confluence como fuente oficial de conocimiento validado.
  - Drive como repositorio documental de apoyo.
  - Jira como sistema de gestión de pendientes y ejecución.

## Lineamientos clave
- Separación estricta entre datos de cliente (A Kantar) y KOVA CORE.
- Human-in-the-loop obligatorio para validar conocimiento y tareas.
- Uso de este repositorio solo para modelos, resúmenes y metadatos,
  evitando almacenar datos personales o sensibles.
